# Требования к проекту "Сервис по рекомендации событий"
В системе есть 3 типа пользователей: администраторы, организаторы и рядовые пользователи. 
Администратор управляет системой и верифицирует данные организаторов. 
Организатор публикует информацию о событиях на своей странице. 
Рядовой пользователь пользуется услугами сервиса, подписываясь на организаторов и выбирая понравившиеся события, а также осуществляя поиск нужных вариантов по интересам вокруг себя.

Есть 2 режима работы:
1. **Просмотр событий вокруг себя**
   * По местоположению и указанным категории события, временному промежутку и радиусу поиска происходит поиск доступных вариантов.
   * Выводится информация о найденных событиях, подходящих под условия поиска.
2. **Отметка интересующих событий**
   * По категории события или его описанию происходит поиск событий.
   * Заинтересовавшие варианты можно добавить в избранные.
   * За определённое время до начала события приходит уведомление пользователю о нём.

1. **Регистрация пользователя**. Любой пользователь должен иметь возможность создания учётной записи путём ввода почты и пароля.
2. **Логин/логаут пользователя**
     * Любой пользователь должен иметь возможность входа в систему при помощи ввода почтового адреса и пароля.
     * Любой пользователь должен иметь возможность выхода из системы в любой момент времени.
3. **(INDEX) просмотр списка записей**. Любой пользователь должен иметь возможность просматривать информацию о событиях.
4. **(CREATE) создание записи**
     * Организатор вводит информацию о событиях.
     * Администратор может создавать записи в БД.
5. **(READ) просмотр деталей записи**
     * Рядовой пользователь может просматривать данные своего профиля, избранное, информацию о событиях.
     * Организатор может просматривать свои данные и информацию о мероприятиях.
     * Администратор имеет доступ к данным всей БД.
6. **(UPDATE) редактирование записи**
     * Все пользователи имеют возможность изменить свои персональные данные.
     * Организатор моржет редактировать информацию о событиях.
     * Администратор способен изменить данные в БД. 
7. **(DELETE) удаление записи**. Администратор способен удалять данные в БД. Кроме того, организатор может удалять/отменять проводимые им ивенты.
